# Checklist: Squeeze Page Quality

**Use when:** Reviewing a lead capture page before launching traffic.

---

## Section 1: Above the Fold (No Scroll Required)

- [ ] Headline communicates the lead magnet benefit clearly
- [ ] Visitor understands what they get within 3 seconds of landing
- [ ] Opt-in form is visible without scrolling
- [ ] Opt-in button text is action-specific ("Send Me the Guide" not "Submit")
- [ ] There is only ONE offer on this page (no navigation, no other links)

## Section 2: Headline

- [ ] Headline is specific (includes number, timeframe, or specific outcome)
- [ ] Headline matches the ad or traffic source headline (message match)
- [ ] Subheadline clarifies or amplifies the headline
- [ ] Headline speaks to the avatar's primary desire, not a feature

## Section 3: Bullets (if present)

- [ ] Each bullet is written with a curiosity-gap structure
- [ ] Bullets reveal benefit without revealing the full content
- [ ] Bullets use the "you'll discover..." or "find out how to..." format
- [ ] 3-5 bullets maximum above the fold (more goes below)
- [ ] No bullet is a generic claim without specificity

## Section 4: Social Proof (for warm or competitive traffic)

- [ ] At least one testimonial or social proof element present (if available)
- [ ] Social proof is specific (name, result, timeframe — no initials)
- [ ] Social proof positioned near or above the opt-in form
- [ ] Download count or member count if substantial ("Join 24,000 readers")

## Section 5: Trust Elements

- [ ] Privacy statement present near email field ("No spam — ever")
- [ ] No unnecessary fields (email only — not name + email + phone for cold traffic)
- [ ] Page loads in under 3 seconds (mobile especially)
- [ ] Mobile opt-in form works correctly

## Section 6: Traffic Temperature Calibration

- [ ] Cold traffic: Page explains what the lead magnet IS (not assumed knowledge)
- [ ] Warm traffic: Page can reference the source ("As seen on [channel/podcast]")
- [ ] Retargeting: Headline can be bolder (they've seen the offer before)

## Section 7: Post-Opt-In

- [ ] Confirmation/thank-you page defined
- [ ] Thank-you page has an immediate action (watch video / read guide / join group)
- [ ] Email confirmation (if double opt-in) is set up
- [ ] Tracking pixel fires on form submission (not on page load)

## Veto Conditions

- VETO if headline is generic ("Free Guide Inside")
- VETO if there are navigation links or external links on the page
- VETO if opt-in button says "Submit" or "Click Here"
- VETO if message doesn't match the traffic source
- VETO if page has multiple offers or competing CTAs
