# Tripwire Page Checklist
**Agent:** `@drm:ryan-deiss`
**Purpose:** Validate Tripwire offer page before launch
**Standard:** Ryan Deiss CVO Framework — Step 4
**Run:** Before publishing any Tripwire offer page

---

## 🔴 CRITICAL — Tripwire Foundation

### Price/Value Equation
- [ ] Tripwire price is $1–$37 range
- [ ] Perceived value is obviously higher than the price (at least 10x)
- [ ] Prospect can immediately understand why this is a deal
- [ ] "Why so cheap?" is addressed somewhere on the page

### Tripwire Purpose Alignment
- [ ] Goal is to create a BUYER, not to profit (profit comes from upsells)
- [ ] Tripwire is directly related to the Lead Magnet consumed before it
- [ ] Tripwire is the natural next step after the Lead Magnet
- [ ] Tripwire creates appetite for the Core Offer (doesn't fully satisfy)

### Irresistibility Test
Ask yourself: "Would I feel stupid NOT buying this at this price?"
- [ ] Yes → proceed
- [ ] No → the value/price gap is insufficient. Fix before launching.

---

## 🟡 HIGH PRIORITY — Page Elements

### Headline
- [ ] Headline leads with the value gap ("Get $[value] for $[price]")
- [ ] OR headline leads with the most irresistible benefit
- [ ] Subheadline clarifies who this is for and what they get

### Body Copy
- [ ] Specific list of what's included (every item with its standalone value)
- [ ] Value stack shows total value before price reveal
- [ ] "Why so cheap?" explanation present (builds trust, reduces skepticism)
  - Acceptable reasons: limited time intro offer, loss leader, mission-driven, etc.
- [ ] Price reveal happens AFTER value is fully established

### Proof Elements
- [ ] At least 2-3 testimonials specific to the Tripwire product
- [ ] Testimonials focus on the quick win the Tripwire delivers

### Order Elements
- [ ] Clear CTA button with action-oriented text ("Yes, I want [specific thing]")
- [ ] Checkout is simple (1-2 fields max visible above fold)
- [ ] Trust badges visible (SSL, guarantee, secure checkout)
- [ ] Guarantee stated clearly on the page

### Order Bump
- [ ] Order bump is present on checkout page
- [ ] Order bump is logically related to the Tripwire
- [ ] Order bump price is 50-100% of Tripwire price
- [ ] Order bump description is one sentence (maximum)
- [ ] Order bump has its own mini-headline on checkout

---

## 🟢 BEST PRACTICE — Optimization

### Post-Purchase Flow
- [ ] Thank-you page is actually an upsell page (Core Offer)
- [ ] The transition copy connects Tripwire to Core Offer naturally
- [ ] Buyer is immediately tagged and moved to buyer email sequence (BDR)
- [ ] Buyer is removed from prospect email sequence

### Mobile Optimization
- [ ] Tripwire page loads in under 3 seconds on mobile
- [ ] CTA button is large enough to tap easily on mobile
- [ ] Value stack is scannable on a small screen
- [ ] Checkout process works without friction on mobile

---

## ❌ AUTO-FAIL CONDITIONS

- [ ] ❌ No "why so cheap?" explanation (creates skepticism, kills conversion)
- [ ] ❌ Price revealed before value is established
- [ ] ❌ Tripwire is unrelated to Lead Magnet (breaks journey)
- [ ] ❌ No order bump on checkout (major revenue leak)
- [ ] ❌ Checkout page asks for too much information
- [ ] ❌ Thank-you page is not an upsell to Core Offer
