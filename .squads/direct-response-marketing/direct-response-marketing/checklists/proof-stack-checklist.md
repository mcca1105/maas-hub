# Checklist: Proof Stack Quality

**Use when:** Reviewing the proof elements before embedding in copy.

---

## Section 1: Testimonial Standards

- [ ] Every testimonial has: full name + location + specific result
- [ ] No testimonial uses only initials or "Anonymous"
- [ ] Every testimonial contains a specific, measurable result (not "it changed my life")
- [ ] Testimonials include a timeframe for results achieved
- [ ] Written permission documented for every testimonial used

## Section 2: Testimonial Diversity

- [ ] Testimonials represent multiple different sub-avatars (not all the same type)
- [ ] At least one testimonial from a skeptic ("I was doubtful at first...")
- [ ] At least one testimonial addressing a common objection
- [ ] At least one testimonial from someone the prospect can identify with
- [ ] Video testimonials prioritized where available

## Section 3: Proof Hierarchy (Sequencing)

- [ ] Scientific/expert proof is positioned first (highest credibility tier)
- [ ] Testimonials follow after establishing credibility
- [ ] Volume/social proof (numbers) appears where critical mass is demonstrated
- [ ] Proof is positioned at the point in copy where the objection naturally arises

## Section 4: Data and Scientific Claims

- [ ] Every study cited has a named institution and publication
- [ ] Every statistic has a source citation
- [ ] Claims are specific (not "studies show" — which studies, by whom, when)
- [ ] Cited research is relevant to the specific claim being made (not tangentially related)

## Section 5: Expert Endorsements

- [ ] Expert's credentials are visible and relevant to the claim
- [ ] Expert quote is specific (not generic "this is great")
- [ ] Permission to use endorsement is documented
- [ ] Expert is genuinely credible in this market (not just generally famous)

## Section 6: Proof Gap Analysis

- [ ] Proof inventory has been mapped to the top 5 objections
- [ ] Each major objection has at least one proof element addressing it
- [ ] Gaps in proof have been identified
- [ ] Plan to acquire missing proof is documented (customer survey / expert outreach / study citation)

## Section 7: Compliance

- [ ] All claims supported by proof match what the product actually delivers
- [ ] Income/results claims include typical results disclosure if required
- [ ] No fabricated or manufactured testimonials
- [ ] All "before/after" images are authentic and unaltered

## Veto Conditions

- VETO if any testimonial uses only initials or has no specific result
- VETO if scientific claims don't have verifiable sources
- VETO if all testimonials are from the same type of person
- VETO if proof is presented before understanding what objections it addresses
- VETO if proof is generic (not specific to the mechanism claimed)
