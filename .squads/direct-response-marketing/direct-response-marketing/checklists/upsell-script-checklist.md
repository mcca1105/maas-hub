# Checklist: Upsell Script Quality

**Use when:** Reviewing an OTO (One-Time Offer) or upsell script.

---

## Section 1: OTO1 Script

- [ ] OTO1 opens with congratulations and validates the core purchase
- [ ] OTO1 immediately identifies the "next problem" (one they'll face after the core purchase)
- [ ] OTO1 is positioned as completing or accelerating the core purchase journey
- [ ] OTO1 price is a natural step up from core (typically 2-5x core price)
- [ ] OTO1 value stack has at least 5:1 ratio vs. price

## Section 2: OTO Sequence Logic

- [ ] OTO2 (if exists) builds on OTO1 — doesn't introduce new problem
- [ ] Downsell path exists for all OTO declines
- [ ] Downsell is a smaller version of OTO (not a completely different product)
- [ ] "No" path (declining buyer) doesn't feel punishing
- [ ] Maximum 3 offers in upsell sequence (beyond that, abandonment spikes)

## Section 3: Script Content

- [ ] Script does NOT repeat all the features of the core product
- [ ] Script opens with what they now HAVE (the core product) and what they'll NEED next
- [ ] Specific result from OTO is stated (not generic "get more results")
- [ ] Social proof specific to OTO is present (not same proof as core)
- [ ] Guarantee is stated for OTO (separate from core guarantee)

## Section 4: Urgency

- [ ] "One-time" claim is genuine (OTO is not regularly available at checkout)
- [ ] If price is genuinely higher outside funnel, that's stated clearly
- [ ] If bonus is fast-action, quantity limit is real and enforced

## Section 5: Technical

- [ ] Video OTO: Script is under 10 minutes for OTO1, under 7 minutes for OTO2
- [ ] Text OTO: Long-form copy follows headline → hook → story → proof → price → CTA
- [ ] "Yes" button copy is active and specific ("Yes! Add [Product] to my order")
- [ ] "No" button copy is not guilt-inducing (avoid "No, I don't want success")
- [ ] Both "yes" and "no" paths confirmed to function technically

## Section 6: Compliance

- [ ] All claims are substantiated and accurate
- [ ] "One-time offer" claim is truthful (not shown at checkout on every purchase)
- [ ] Results claims follow FTC guidelines (typical results disclaimer if needed)

## Veto Conditions

- VETO if OTO introduces a completely different problem from the core purchase journey
- VETO if there's no downsell (forcing all-or-nothing loses revenue)
- VETO if "No" button language shames the buyer
- VETO if OTO claims to solve the SAME problem as the core (cannibalization)
- VETO if sequence has more than 3 offers before order confirmation
