# Checklist: Full Copy Review

**Use when:** Conducting final review of any major copy asset (VSL, sales letter, landing page).

---

## Section 1: Specificity Audit (Gary Halbert Standard)

- [ ] Every number in the copy is specific (not "many" or "thousands" — exact counts)
- [ ] Every timeframe is specific ("in 14 days" not "quickly")
- [ ] Every name mentioned is a real, full name (first + last)
- [ ] Every result claimed is a specific, verifiable result (not "amazing results")
- [ ] Specificity ratio: Count specific claims vs. vague claims — target 80% specific

## Section 2: Structure Integrity

- [ ] Lead section establishes the problem BEFORE introducing the solution
- [ ] Mechanism is explained before the product is named
- [ ] Proof comes before price revelation
- [ ] Guarantee appears near the price and CTA
- [ ] P.S. (if present) contains urgency + restatement of core offer

## Section 3: Avatar Language Match

- [ ] Copy uses the avatar's exact vocabulary (not marketing-speak)
- [ ] Copy mirrors the phrases from VOC research
- [ ] The problem is described the way the avatar describes it (their label, not ours)
- [ ] Copy avoids jargon the avatar wouldn't use
- [ ] Reading copy aloud — does it sound like someone talking to the avatar, or at them?

## Section 4: Objection Coverage

- [ ] Price objection handled (value reframe + cost of inaction)
- [ ] "Will this work for me?" objection handled (identity-matching testimonial)
- [ ] "I've tried similar things before" objection handled (mechanism differentiation)
- [ ] "Now isn't the right time" objection handled (cost of delay)
- [ ] Risk objection handled (guarantee)

## Section 5: Persuasion Flow

- [ ] Desire is amplified before the solution is offered
- [ ] Each section flows logically to the next (no non-sequiturs)
- [ ] Copy doesn't go from problem → solution without establishing WHY the solution works
- [ ] Proof is matched to the specific claim it supports (not generic testimonials after unrelated claims)

## Section 6: Technical Review

- [ ] All links tested and working
- [ ] Copy doesn't start sentences with "I" (rule of thumb for copy — you-focused)
- [ ] Active voice used throughout (not "results can be achieved")
- [ ] Paragraph length varies (no blocks of more than 4 lines)
- [ ] Formatting aids readability (subheads, bullets, bold for key phrases)

## Section 7: Legal & Compliance

- [ ] All income claims include typical results disclosure
- [ ] Testimonials are genuine and permission is documented
- [ ] Before/after images are authentic and not misleading
- [ ] Claims don't make medical/legal/financial promises without qualification
- [ ] FTC compliance standards met for the claim category

## Final Gut-Check Questions

- [ ] Would this copy make Gary Halbert embarrassed by its lack of specificity? (NO required)
- [ ] Is there a single line that could be cut without losing something? (If yes, cut it)
- [ ] Does reading the first 50 words make you want to read the next 50? (YES required)
- [ ] Is this copy talking to one person or to a crowd? (Should be ONE person)

## Veto Conditions

- VETO if specificity ratio is below 50% (too many vague claims)
- VETO if solution is introduced before the problem is fully established
- VETO if price is revealed before the value anchor
- VETO if any of the 5 core objections are not addressed
- VETO if copy reads as corporate (formal language, passive voice, "we are pleased to")
