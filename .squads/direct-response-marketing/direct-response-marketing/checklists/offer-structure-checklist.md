# Checklist: Irresistible Offer Structure

**Use when:** Reviewing an offer before writing copy or launching.

---

## Section 1: The Core Promise

- [ ] The offer has one clear, specific, measurable promise (not 3 vague promises)
- [ ] The promise is a result/outcome, not a feature ("Lose 20 lbs" not "Meal plan included")
- [ ] A reasonable person can achieve this promise if they use the product
- [ ] The timeframe for the promise is realistic and specified
- [ ] The promise addresses the avatar's primary desire (from VOC research)

## Section 2: Value Stack

- [ ] Every component in the value stack has a name (not "Module 1")
- [ ] Every component has a stated dollar value with justification
- [ ] Dollar values are defensible (based on comparable market rates)
- [ ] Total value stack is at least 5x the asking price (10x is ideal)
- [ ] At least one bonus is time-limited or quantity-limited (fast-action incentive)

## Section 3: No-Brainer Test (4 Questions)

- [ ] Would this be an easy yes if the price were $1? (YES required)
- [ ] Is the value-to-price ratio at least 10:1 in the buyer's perception? (YES required)
- [ ] Does every component directly help achieve the core promise? (YES — or remove it)
- [ ] Is it embarrassing how much value is in this deal? (YES required)

If any NO → Review the value stack and promise before writing copy.

## Section 4: Guarantee

- [ ] The guarantee has a specific, memorable name
- [ ] The guarantee terms are specific (what happens, how, when)
- [ ] The claim process is simple and clearly described
- [ ] Risk is fully reversed — buyer has nothing to lose
- [ ] Guarantee is positioned at 2 points in copy (first mention + near CTA)

## Section 5: Unique Mechanism

- [ ] The offer has a named, unique mechanism
- [ ] The mechanism explains WHY results happen (not just what it is)
- [ ] The mechanism differentiates from competitors (not generic "system" or "method")
- [ ] The mechanism name is memorable and ownable
- [ ] The mechanism is understood by the buyer, not just the seller

## Section 6: Price Presentation

- [ ] Anchor is established before price is revealed
- [ ] Anchor is at least 3x the actual price
- [ ] A genuine reason for the discount/price is stated
- [ ] Payment plan (if offered) is slightly more than single payment
- [ ] Cost of inaction is quantified in the copy

## Veto Conditions

- VETO if offer promise is vague ("transform your life")
- VETO if value stack total is less than 3x the asking price
- VETO if price is revealed before establishing anchor value
- VETO if guarantee requires complex process to claim
- VETO if no unique mechanism named
