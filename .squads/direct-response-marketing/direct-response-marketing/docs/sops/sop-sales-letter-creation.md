## Phase 1: Market and Prospect Research

### The Halbert List Research Protocol

Before writing, rank your prospect list using the RFU formula:
1. **Recency** — When did they last buy or respond?
2. **Frequency** — How often do they buy?
3. **Unit of Sale** — How much do they spend?

**Write to your best buyer first.** The person who has bought recently, buys often, and spends the most is your ideal prospect. Write the letter to them.

### VOC Research Checklist
- [ ] Spend 2+ hours reading the prospect's words (forums, reviews, social)
- [ ] Extract 20+ exact phrases they use to describe their problem
- [ ] List 5 previous solutions they've tried
- [ ] Identify the specific frustration with each failed solution
- [ ] Find the dream outcome in their exact words
- [ ] Identify their #1 objection to buying

### The Nugget Notes Process (Halbert's Research System)
- [ ] Write each prospect frustration on a separate index card (or line)
- [ ] Write each desire on a separate card
- [ ] Write each objection on a separate card
- [ ] Write each proof element on a separate card
- [ ] Write potential hooks/stories on separate cards
- [ ] Arrange until a natural narrative flow emerges

## Phase 2: Headline Development

**Rule:** Write the letter's headline LAST. Or write it first as a hypothesis and rewrite it after completing the body.

Write at least 25 headline options before choosing. Use these categories:

### Category 1: Curiosity Headlines
Create an information gap that demands resolution.
```
"The Strange [Discovery/Trick/Secret] That [Result] Without [Sacrifice]"
"What Your [Doctor/Boss/Bank] Doesn't Want You to Know About [Topic]"
"Why [Common Belief] Is Wrong (And What Actually Works)"
```

### Category 2: News/Announcement Headlines
Frame as a genuine discovery or announcement.
```
"Announcing: A New [Product/Method] That [Claim] in [Timeframe]"
"Finally: [Problem] Solved by [Unexpected Source]"
"New Study Reveals: The Real Cause of [Problem]"
```

### Category 3: How-To Headlines (Warm/Hot Traffic)
Use only when prospect already knows the category of solution.
```
"How to [Result] in [Timeframe] Even If [Common Objection]"
"How [Specific Person] [Result] Using [Method]"
```

### Category 4: Command Headlines
Direct, for audiences that respect authority.
```
"Read This Before You [Common Action Related to Problem]"
"Give Me [X Days/Minutes] and I'll Prove [Bold Claim]"
```

### Headline Selection Criteria
- [ ] Does it make a specific promise?
- [ ] Does it create curiosity or communicate benefit?
- [ ] Is it written at the right awareness level?
- [ ] Does it select FOR the right prospect and AGAINST the wrong one?
- [ ] Would you stop scrolling if you saw it?

## Phase 3: Letter Body — The 28 Elements

Write the complete letter using this sequence. Not all elements appear in every letter — use judgment based on offer complexity and awareness stage.

**OPENING SECTION**

```
[01] HEADLINE — The primary attention device
[02] SUBHEADLINE — Expands or qualifies the headline
[03] SALUTATION — "Dear Friend," (never "Dear Valued Customer")
[04] HOOK SENTENCE — First sentence designed only to get the second sentence read
```

**PROBLEM SECTION**

```
[05] THE PROBLEM STATEMENT — Name the problem in the prospect's words
[06] PROBLEM AGITATION — Expand on consequences, make pain vivid
[07] EMPATHY STATEMENT — "I know exactly how you feel, because I felt the same way"
[08] CREDIBILITY SETUP — Why you're positioned to have this conversation
```

**SOLUTION SECTION**

```
[09] THE PIVOT — "But then I discovered..."
[10] SOLUTION TEASE — Glimpse of the result without full reveal
[11] STORY/CREDIBILITY — The full story of discovery or transformation
[12] MECHANISM INTRODUCTION — Why this solution works differently
[13] MECHANISM PROOF — Evidence supporting the mechanism
```

**OFFER SECTION**

```
[14] PRODUCT INTRODUCTION — Name and what it is
[15] PRODUCT BENEFITS — What they get with each feature
[16] PROOF STACK — Testimonials, data, case studies
[17] OBJECTION HANDLING — Address top objections directly
[18] OFFER REVEAL — What they receive (everything)
[19] VALUE STACK — Individual prices of each component
[20] PRICE ANCHOR — Higher reference prices before your price
[21] PRICE REVEAL — Your actual price
[22] PAYMENT OPTIONS — If applicable (monthly plan)
[23] GUARANTEE — Terms, duration, how it works, risk reversal
[24] URGENCY — Real reason to act now
[25] SCARCITY — Real reason for limited availability
```

**CLOSE SECTION**

```
[26] CALL TO ACTION — Specific instruction, what to do right now
[27] CONSEQUENCE OF INACTION — What they lose by not acting
[28] P.S. — Best benefit + guarantee + deadline restated
```

## Phase 4: Fascination Bullets

Bullets are mini-ads within the letter. Each one should make the prospect think "I need to know this."

### The Mel Martin Bullet Formula
```
"[What it is] ... but not [what they'd expect]"
```

### Bullet Writing Rules
- [ ] Every bullet promises specific, actionable information
- [ ] Never reveal the full answer in the bullet
- [ ] Use odd numbers and page references ("Page 47 reveals...")
- [ ] Test bullets as if they're standalone headlines

### Bullet Categories to Cover
- "The one thing you must never do if you want to [desired outcome]"
- "The counterintuitive [method/ingredient/step] that [result]"
- "Why [common advice] is exactly wrong — and what to do instead"
- "The [specific page] on which [specific revelation]"
- "How to [achieve small specific result] in [short timeframe]"
- "The [specific item/tool] that [specific expert] uses every day"

**Bullet Count:** Aim for 15-30 bullets per sales letter.

## Phase 5: Guarantee Copy

The guarantee is not a formality. It's one of the most important selling elements.

### Guarantee Principles (Halbert)
- Transfer ALL risk to the seller
- State it boldly, not defensively
- Make the terms so clear they remove every possible objection

### Guarantee Formula
```
"Try [Product] for [Duration] — completely at our risk.
If you're not [specific result] in [timeframe],
simply [how to return] and we'll [refund terms].
No questions asked. No forms to fill out. No hassle.
You risk nothing. The burden is entirely on us."
```

### Strong vs. Weak Guarantees

**Weak:** "30-day money-back guarantee"
**Strong:** "Use [Product] for the next 60 days. If you don't [specific result], send it back for a full refund — even if you've used every last [unit]. Keep all the bonuses as our thank-you for trying."

## Phase 6: P.S. Section

The P.S. is the second-most-read element after the headline. Many prospects skip directly to it.

The P.S. must contain:
1. The single biggest benefit (the most compelling promise)
2. The guarantee restated briefly
3. The urgency/deadline restated
4. The CTA (click, call, or order)

**P.S. Template:**
```
P.S. Remember: [Biggest promise]. Plus, you're covered by our [guarantee term]
guarantee — so you truly have nothing to lose and everything to gain.
But [urgency/scarcity reason], so [CTA] right now.
```

## Phase 7: The A-Pile Test (Direct Mail)

If this is a direct mail letter, apply the A-Pile test:

- [ ] Real stamp (not meter)
- [ ] No corporate logo on envelope
- [ ] Handwritten or personal-looking font for addressing
- [ ] No promotional language visible on envelope exterior
- [ ] First class postage (not bulk)

**Digital Equivalent:**
- [ ] Email subject line looks personal ("Quick question for you")
- [ ] Sender name is a person, not a company
- [ ] No company logo in subject line
- [ ] Preview text creates curiosity

---

## Phase 8: The WIIFM Audit

After writing the full letter, run the "What's In It For Me" audit:

For each paragraph, ask: **What does the READER get from reading this?**

If the answer is "nothing — it's about us/the product/the company":
- Delete the paragraph, OR
- Reframe it from the reader's perspective

**Count check:**
- [ ] Count "you" and "your" in the letter
- [ ] Count "we", "our", "I", "me"
- [ ] "You" should outnumber "we/I" by at least 3:1

## Quality Control Checklist

**Headlines:**
- [ ] 25+ headlines written before selecting
- [ ] Selected headline passes the "would I stop scrolling" test
- [ ] Subheadline supports or expands the headline

**Copy:**
- [ ] Average sentence length under 15 words
- [ ] Paragraphs are 1-4 lines max
- [ ] Reading level: 6th-8th grade (check with readability tool)
- [ ] No passive voice
- [ ] All claims are specific (numbers, names, places, dates)

**Offer:**
- [ ] Value stack totals are believable
- [ ] Guarantee is bold and transfers all risk
- [ ] Urgency/scarcity is real and explained
- [ ] P.S. contains: benefit + guarantee + urgency + CTA

**Structure:**
- [ ] AIDA sequence is intact
- [ ] Objections are addressed before they become deal-breakers
- [ ] Transitions carry reader from section to section
